class Head(nn.Module) — Scaled Dot-Product Attention

Datei: model.py, Zeile 27 · Ein kausaler Single-Head Attention-Mechanismus

Was macht ein Attention-Head?

Ein Attention-Head entscheidet für jede Token-Position im Eingabe-Tensor, welche anderen Positionen wie stark relevant sind, und erzeugt daraus einen neuen, kontextangereicherten Vektor. Im Decoder-Only-Modell (GPT-Stil) darf dabei jede Position nur in die Vergangenheit schauen — das erzwingt die kausale Maske.

1. Initialisierung — __init__

Vier Objekte werden registriert:

self.key   = nn.Linear(n_embd, head_size, bias=False)  # W_K
self.query = nn.Linear(n_embd, head_size, bias=False)  # W_Q
self.value = nn.Linear(n_embd, head_size, bias=False)  # W_V

self.register_buffer("tril", torch.tril(torch.ones(block_size, block_size)))
self.dropout = nn.Dropout(dropout)
K
key — "Was biete ich an?" Projiziert jeden Eingabe-Vektor auf einen head_size-dimensionalen Key-Vektor. Lernbare Gewichtsmatrix W_K ∈ ℝ^(n_embd × head_size).
Q
query — "Was suche ich?" Projiziert jeden Eingabe-Vektor auf einen Query-Vektor. Lernbare Gewichtsmatrix W_Q ∈ ℝ^(n_embd × head_size).
V
value — "Was gebe ich weiter?" Enthält die eigentliche Information, die bei Relevanz weitergegeben wird. Lernbare Gewichtsmatrix W_V ∈ ℝ^(n_embd × head_size).
▽
tril — Untere Dreiecksmatrix (nicht lernbar) Ein fester Buffer (kein Gradient). Blockiert den Blick in die Zukunft. register_buffer stellt sicher, dass er beim .to(device)-Aufruf mitgenommen wird.

2. Forward-Pass — Schritt für Schritt

B, T, C = x.shape   # Batch-Größe | Sequenzlänge (Zeitschritte) | Kanäle (n_embd)

Der Eingabe-Tensor x hat die Form (B, T, C) — B parallele Sequenzen, jede mit T Tokens, jedes Token als C-dimensionaler Vektor.

Schritt 1 — Projektion auf Q, K, V

k = self.key(x)    # (B, T, head_size)
q = self.query(x)  # (B, T, head_size)
v = self.value(x)  # (B, T, head_size)

Jedes der T Tokens bekommt einen eigenen Key-, Query- und Value-Vektor durch die gelernten Projektionen.

Schritt 2 — Attention-Scores (Scaled Dot-Product)

wei = Q · Kᵀ · (head_size)^(−½)
Form: (B, T, T) — für jedes Token-Paar (i, j) ein Score 
wei = q @ k.transpose(-2, -1) * (head_size ** -0.5)   # (B, T, T)

Das Skalarprodukt von Query i mit Key j misst, wie relevant Token j für Token i ist. Die Skalierung mit 1/√head_size verhindert, dass die Werte bei großem head_size so groß werden, dass Softmax gesättigte Gradienten produziert.

Schritt 3 — Kausale Maske (Decoder-Eigenschaft)

wei = wei.masked_fill(self.tril[:T, :T] == 0, float("-inf"))

Alle Positionen oberhalb der Hauptdiagonale (Zukunft) werden auf -∞ gesetzt. Nach Softmax werden diese zu exakt 0 — das Modell "sieht" sie nicht.

Intuition: Token 0 sieht nur sich selbst. Token 3 sieht alle 4 Tokens. Kein Token kann in die Zukunft schauen — das ist die autoregressive Eigenschaft des Decoders.

Schritt 4 — Softmax & Dropout

wei = F.softmax(wei, dim=-1)  # Gewichte summieren sich pro Zeile auf 1
wei = self.dropout(wei)         # Regularisierung: zufällig Connections kappen

Softmax normiert die Scores zeilenweise zu Wahrscheinlichkeiten. Dropout setzt zufällig einige Attention-Gewichte auf 0 (nur beim Training), was Überanpassung reduziert.

Schritt 5 — Gewichtete Summe über Values

output = wei · V
Form: (B, T, head_size) 
return wei @ v

Jedes Output-Token ist eine gewichtete Summe aller (erlaubten) Value-Vektoren. Hohe Attention-Gewichte bedeuten: "Dieser Value-Vektor trägt stark zu meiner Ausgabe bei."

Die vollständige Formel

Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) · V
dₖ = head_size  ·  Masking der oberen Dreiecke vor Softmax

Datenfluss auf einen Blick

x  (B, T, n_embd)
 │
 ├── W_Q →  Q  (B, T, head_size)  ─┐
 ├── W_K →  K  (B, T, head_size)   ├─ Q·Kᵀ/√dₖ → wei (B, T, T)
 │                                 ┘     │
 │                                   masked_fill(-∞) + softmax + dropout
 │                                       │
 └── W_V →  V  (B, T, head_size)  ◄─ wei @ V
                                         │
                                   output (B, T, head_size)

Einbettung in MultiHeadAttention

Die Klasse MultiHeadAttention erstellt n_heads parallele Instanzen von Head, jede mit eigenem W_Q, W_K, W_V. Jeder Head lernt andere Aspekte der Beziehungen zwischen Tokens (z.B. Syntax, Semantik, Position). Die Ausgaben aller Heads werden konkateniert und durch eine lineare Projektion auf n_embd zurückgeführt. 

# In MultiHeadAttention.forward:
out = torch.cat([h(x) for h in self.heads], dim=-1)  # (B, T, n_heads * head_size)
return self.dropout(self.proj(out))                    # (B, T, n_embd)
